变量和数据类型

声明变量​

// 声明变量a 值为 字符串 666 , 分号可加可不加 , 单引号和双引号作用一样
var a = '666';
let name = tom;
const measurements  = [92,60,88];

var、let和const​

var 为 ES5 前唯一 声明变量的方式 , 而let 和const 是ES6推出的全新声明变量的方式, 用以代替var的致命缺陷

var的致命缺陷
  1. 变量提升
  2. 可以被重复声明
  3. 没有块作用域
  4. 会导致意外的全局变量
  • var
  • let
  • const
var a = 10;
var a = 'js' // 合法
let a = 10;
let a = 5// error!
const π = 3.14159,
π = 3 // error!

const Cat = {id:1,name:'tom'}
Cat.name = 'jerry',

块级作用域​

ES5 只有全局作用域和函数作用域，没有块级作用域，这导致很多场景不合理。

没有块级作用域有什么问题?
  1. 内层变量可能会覆盖外层变量

以下代码的原意是， if 代码块的外部使用外层的 tmp 变量，内部使用内层的 tmp 变量。但是，函数 fn 执行后，输出结果为 undefined ，原因在于因为 if (false) {var tmp = 'hello world'}尽管被{}包裹,但var不存在块级作用域,就会变量提升导致内层的 tmp 变量覆盖了外层的 tmp 变量。

var tmp = new Date()
function fn() {
  console.log(tmp)
  if (false) {
    var tmp = 'hello world'
  }
}
fn() // undefined
  1. 用来计数的循环变量泄露为全局变量
var s = 'hello'
for (var i = O; i < s.length; i++) {
  console.log(s[i])
}
console.log(i) // 5

上面的代码中，变量i只用来控制循环，但是循环结束后，它并没有消失，而是泄露成了全局变量。 let 实际上为 JavaScript 新增了块级作用域。

function fl() {
  let n = 5
  if (true) {
    let n = 10
  }
  console.log(n) // 5
}

上面的函数有两个代码块，都声明了变量 n，运行后输出 5 。这表示外层代码块不受内层代码块的影响。如果使用 var 定义变量 ，最后输出的值就是 10

经典面试题
for (var i = 0; i < 5; i++) {
  setTimeout(() => {
    console.log(i)
  }, 1000)
}
// 5 5 5 5 5

改成 ES6 中的 let

for (let i = 0; i < 5; i++) {
  setTimeout(() => {
    console.log(i)
  }, 1000)
}
// 0 1 2 3 4

这是因为setTimeout是异步代码，会先执行主线程的代码，当主线程代码执行完后，这时候i的值已经是5了，setTimeout便会打印出5个5 而使用let后，无论是()还是{} 都会产生块级作用域,由于块级作用域的特性，每次for循环执行，let声明的变量都会重新绑定一次，i 的值自然是每次执行的值

那么 ES5 能不能实现 块级作用域 的效果呢? 可以的，我们可以利用闭包

for (var i = 0; i < 5; i++) {
  ;(function (index) {
    setTimeout(() => {
      console.log(index)
    }, 1000)
  })(i)
}
// 0 1 2 3 4

变量提升​

提升指的是声明会提升至当前作用域的顶端, 而赋值留在原处
  • 函数声明提升优先于变量提升
  • Javascript中的所有声明均存在提升, 包括var,let,const,class,function,但ES6推出的会存在TDZ(暂时性死区),而表现得像不会提升一样
  • 只有函数声明才有函数提升, 函数表达式是不会提升的
console.log(a) // undefined
var a = 12

实际上提升后的代码
var a;  //提升到作用域顶端，默认赋值undefined
console.log(a) // undefined
a = 12
暂时性死区-在未赋值禁止访问 (temporal dead zone，简称TDZ)

通过 let,const 等声明的变量直到它们的定义被执行时才初始化。在变量初始化前访问该变量会导致 ReferenceError。该变量处在一个自块顶部到初始化处理的“暂时性死区”中

console.log(a)
let a = 12 //ReferenceError: a is not defined

数据类型​

JS数据类型分为: 基本数据类型(七种)和引用类型(一种)，共八种类型
其中symbol为es6、bigint 为es10后新增的数据类型

基本数据类型(7种): 原始值​

  • string (字符串)
  • number (数值)
  • boolean (布尔值)
  • null (空)
  • undefined (未定义)
  • symbol(标志)
  • bigint(大数)

引用数据类型(1种): 引用值​

  • object

堆和栈​

  • 栈
    • 自动分配大小固定的内存空间，自动释放，遵循先进后出原则,存放基本数据类型以及指向存放在堆中引用类型的地址指针。
  • 堆
    • 大小不固定，存放引用类型，保持引用则垃圾回收不会回收。

基本类型采用的是值传递，而引用类型是地址传递。结合下图理解，就可以知道，如果找一个基本类型是直接去栈中找，而找一个引用类型，则会先去找它的地址指针(存放在栈中)，然后通过这个地址找到在堆内存的位置从而找到引用类型

深浅拷贝​

  • 浅拷贝
    • 浅拷贝只复制指向某个对象的指针，而不复制对象本身，新旧对象还是共享同一块内存。但深拷贝会另外创造一个一模一样的对象，源对象与拷贝对象互相独立，新对象跟原对象不共享内存，修改新对象不会改变旧对象。
  • 深拷贝
    • 深拷贝和浅拷贝都是针对引用类型数据，因为基本数据类型赋值后（开辟新内存空间存放新值），改变新数据，都不会影响到原来的数据。也就是说对于基本数据类型来说，无论深浅拷贝，数据都是共用的，只要一个拷贝的值发生改变，另外一个不会受影响。

浅拷贝：​

直接赋值、 object.assign、 concat() 、slice()、 Array.from() ...拓展运算符 ，以上都是浅拷贝，除了直接赋值，如果只涉及基本数据类型和单层引用类型，你也可以认为他们是(伪)深拷贝。

如果记不住, 那只要记住不是深拷贝(就2种)都是浅拷贝就得了~

深拷贝：​

  • JSON.parse(JSON.stringify(object))
  • 手写深拷贝
    • 更好的处理方式当然是手写深拷贝了，但手写深拷贝需要处理许多问题，很多人写的其实都没有考虑周到，比如原型链如何处理、DOM 如何处理等等，推荐使用lodash 等工具库的深拷贝函数。
JSON这种方法虽然快速，但有几个缺点。如
  • 会忽略 undefined
  • 会忽略 symbol
  • 不能序列化函数
  • 不能解决循环引用的对象

浮点数精度丢失​

JS 的数字类型是浮点类型的，没有整型。并且浮点类型基于 IEEE 754标准实现（64位），基于该标准的语言都存在浮点数运算精度丢失的问题(PHP,Java，Python等)。

因为计算机只认识二进制,也就是0和1, 在转换浮点数如0.1的时候, 二进制是无限循环小数, 而受限于精度位数原因, 只能截取53位小数, 所以会存在误差,也就是 浮点数精度丢失的问题 , 你可以测试 0.1 + 0.2 !== 0.3

判断类型​

typeof​

typeof 判断基本数据类型, 返回该类型的字符串
  • 可以判断基本数据类型，除了Null (null是object，这是1个设计上的BUG)
  • 不能判断引用类型，因为都统一返回object
typeof 666 // 'number'
typeof '0' // 'string'
typeof undefined // 'undefined'
typeof true // 'boolean'

const sym1 = Symbol();
typeof sym1 // 'symbol'

const HugeNumber = BigInt(9007199254740991);
typeof HugeNumber //"bigint"

typeof null //'object' 基本数据类型除了null都是判断准确的。这情况是JS设计的一个缺陷
typeof NaN //'number' NaN = not a number 所以NaN是Number类型，JS真有你的哦~

typeof并不能准确判断引用类型，因为除了函数外返回的都是'object'
typeof function(){}  //'function'
typeof [1,2,3] //'object'

instance of​

A instanceof B 布尔值，true 或false
  • 可以判断引用类型判断引用类型,检测构造函数的 prototype 属性是否出现在该实例对象的原型链上
// 定义构造函数
function C(){}
function D(){}

var o = new C();

o instanceof C; // true，因为 Object.getPrototypeOf(o) === C.prototype
o instanceof D; // false，因为 D.prototype 不在 o 的原型链上
o instanceof Object; // true，因为 Object.prototype.isPrototypeOf(o) 返回 true
C.prototype instanceof Object // true，同上

C.prototype = {};
var o2 = new C();

o2 instanceof C; // true
o instanceof C; // false，C.prototype 指向了一个空对象,这个空对象不在 o 的原型链上.

D.prototype = new C(); // 继承
var o3 = new D();
o3 instanceof D; // true
o3 instanceof C; // true 因为 C.prototype 现在在 o3 的原型链上

Object.prototype.toString().call()​

这是判断类型最为准确的方法，能够准确判断基本数据类型(包括null)和引用类型的方法
  • 为什么不直接toString(XX)呢？ toString() 方法返回一个表示该对象的字符串，call()让this指向该对象。
    • 每一个继承 Object 的对象都有 toString 方法,对于大多数对象(Array、Date、RegExp等)继承自Object的，toString() 方法都是重写了的，这时就需要用 call() 或 Reflect.apply() 等方法来调用。

我们可以直接调用Object原型上未被覆盖的toString()方法，使用call来改变this指向来达到我们想要的效果。

// Boolean 类型，tag 为 "Boolean"
Object.prototype.toString.call(true);            // => "[object Boolean]"

// Number 类型，tag 为 "Number"
Object.prototype.toString.call(1);               // => "[object Boolean]"

// String 类型，tag 为 "String"
Object.prototype.toString.call("");              // => "[object String]"

// Array 类型，tag 为 "String"
Object.prototype.toString.call([]);              // => "[object Array]"

// Arguments 类型，tag 为 "Arguments"
Object.prototype.toString.call((function() {
  return arguments;
})());                                           // => "[object Arguments]"

// Function 类型， tag 为 "Function"
Object.prototype.toString.call(function(){});    // => "[object Function]"

// Error 类型（包含子类型），tag 为 "Error"
Object.prototype.toString.call(new Error());     // => "[object Error]"

// RegExp 类型，tag 为 "RegExp"
Object.prototype.toString.call(/\d+/);           // => "[object RegExp]"

// Date 类型，tag 为 "Date"
Object.prototype.toString.call(new Date());      // => "[object Date]"

// 其他类型，tag 为 "Object"
Object.prototype.toString.call(new class {});    // => "[object Object]"

constructor 属性​

constructor 属性返回所有 JavaScript 变量的构造函数。
"John".constructor                 // 返回函数 String()  { [native code] }
(3.14).constructor                 // 返回函数 Number()  { [native code] }
false.constructor                  // 返回函数 Boolean() { [native code] }
[1,2,3,4].constructor              // 返回函数 Array()   { [native code] }
{name:'John', age:34}.constructor  // 返回函数 Object()  { [native code] }
new Date().constructor             // 返回函数 Date()    { [native code] }
function () {}.constructor         // 返回函数 Function(){ [native code] }
