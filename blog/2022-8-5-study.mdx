## 学习方法

> 好的学习方法能让人事半功倍, 如果对学习效率不满意, 不妨判断一下自己的学习方法是否适合自己, 是否足够高效. 笔者也是从不会学习 到 逐渐 培养起适合自己的一套高效学习方法, 希望能够帮助大家.

## 语言学习

从大学才开始接触编程，第一门编程语言是 C，后续接连学习了 JavaScript、Java、Python、Rust 和 等其他语言。语言不在多而在精, 精通一门语言才是重中之重, 但可以学习其他语言来开拓视野,我也来分享下学习这些语言的想法和感受:

- `C` 学习的第一门语言, 从底层出发了解整个计算机系统, 但不好驾驭
- `JavaScript`目前吃饭的家伙,浏览器只能跑 JS,没什么好说的,又爱又恨吧~
- `Java` 生态最成熟应用最广泛, OOP 特性让团队合作更加高效
- `Python` 各种各样的库很多，可以快速验证自己的想法
- `Rust` 学习曲线十分陡峭, 不过好在`编译器会手把手教我写代码`

## 找准方向

### 学习路线

:::info 打好基本功

无论学习什么,一份学习路线的指导是非常有必要的,这样可以让你少走许多弯路,用最短的时间学习最多的知识, 千万不要跳,一个阶段一个阶段的走下去,这是打基本功的好办法, `基础不牢地动山摇`.

:::

### 勤做笔记

:::tip 学习应该伴随 `输入` 和 `输出`

学习一门新语言 or 新技术 一定要`输出`!!! 也许你看了某个视频/文章 你觉得你会了, 但是这些并没有让你大脑留下深刻印象, 因为学习需要输入和输出的反馈, 最好的方法就是 边学边做(动手实现并记录笔记)

:::

### 检验成果

:::note 项目验收

赶紧把你学习的新技术用在项目上吧, 哪怕只是一个简单的 demo, 这也会让你留下更加深刻的印象.

:::

做完项目最好上传到 github 并持续打磨后开源, 如果你是和小伙伴一起完成的那更加好。这不仅让你体会到 git 多人开发的过程，期间你还可以练习如何 pr，如何提交规范的 commit，遇到冲突如何解决，最重要的是你参与了一个开源项目的诞生/贡献！在这项目过程中，你也可以把你学习到的技术运用其中，如果能够获得 1000 个 star，那将是一件了不起的事情！
