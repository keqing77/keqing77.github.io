# 文档

官方文档是最好的教程，没有之一。一个框架/技术，技术人员怎么使用一定会写在文档里，当然最好是英文的，为什么呢？ 因为大多数框架/技术都是老外的，文档肯定是英文的，中文文档有2个问题就是同步不及时和翻译可能会带来歧义，最重要的是英文文档能提高你的英文阅读能力；而英文的学习资源无论是数量上还是质量上，都是碾压中文的。这并不是说中国开发者不如外国开发者，相反中国开发者非常优秀，可以说互联网玩家只有中美两个大国，我这么说的原因是国内技术氛围和中文社区实在不堪入眼。(CSDN满大街水文和抄袭、百度为首的广告搜索引擎)，国内社区更多是以流量为驱动，流量为王，非常浮躁，Github上开源的项目大多数也是KPI项目骗完⭐️后就不维护了；而国外社区则更多是技术驱动，这就注定两者之间的差别，

# B站视频、慕课网

这里我就不推荐什么视频了，这个太多了，只要你想找肯定能找的到的。这里可以建议跟着视频做一两个完整的项目来检验自己的学习成果，但切忌要在掌握基础知识前，你直接跟着做是没有意义的，一口是吃不成大胖子的。其实视频可能看得更多的是某些知识点的解说视频，这些会更深入的去理解，毕竟到了中后期你会觉得看视频非常的浪费时间，非常的啰嗦，这也说明你的水平到了某个层次。

看视频要注意的几个点

  • 要敲代码，先看在敲也行，边看边敲也行，总之要动手，不光看！!
  • 看视频前最好看发布时间，以及视频内容是否还是主流，别学了过时的技术还当宝。(jquery)
  • 最好按着完整的路线去看，因为知识是连贯的，你不会JavaScript还想直接学Vue/React？

# 书籍

# CSS

  • 《CSS揭秘》
  • 《CSS权威指南》

这里推荐的是《CSS揭秘》，这是一本让你卧槽，CSS还能这么的写的书，能让你用CSS实现许多可能你之前认为得JS才能实现的效果，而且注重练习，对CSS的提高非常有帮助，这就是我对这本书的最高评价。 而《CSS权威指南》则是一本工具书，更想你不会就查一下的百科全书，书的内容大致都是介绍属性、怎么用，非常官方的描述。当然可能查阅MDN可能更方便😄

# JS

  • 《JavaScript高级程序设计》
  • 《你不知道的JavaScript》
  • 《JavaScript忍者秘籍》
  • 《JavaScriptDOM编程艺术》

《JavaScript高级程序设计》，人称红宝书，前端人手必备人手一本，经典入门书籍无需多言，门槛低。
《你不知道的JavaScript》，人称小黄书，上中下三册，推荐只看三册，这套书是进阶版，需要有一定JS
《JavaScript忍者秘籍》,jQuery作者写的，名字有点中二
《JavaScriptDOM编程艺术》，尽管现在的框架都不需要你直接操作DOM，但DOM的操作依然是每位前端的基本功。 其他的书就不一一介绍了，暂时没有比上面这四本更好的，诸如犀牛书等等，红宝书就能代替了，无需要重复。

还有一本比较有趣的是- 《JavaScript百炼成仙》，这是编程届最火的修仙教程书了，写的非常有趣，我本来打算只看一章，结果看入迷了，有可能是讲得最好的JS入门书，框架篇就讲得一般了。

# 计算机网络

  • 《图解Http》
  • 《图解TCP/IP》
  • 《Http权威指南》

图解系列图文并茂，适合入门，其实已经很够用了，如果还有进一步钻研，那么《Http权威指南》无疑是经典中的经典。

# 数据结构与算法

  • 《算法》(第四版)
  • 《学习JavaScript数据结构与算法》(第三版)
  • 《算法导论》
  • 《代码随想录》同名书籍 (opens new window)

前端的算法还是尽可能用js或者ts写吧，如果你不限定语言，那么《算法》(第四版)是非常不错的选择，是java实现的。当然还有很多...
《学习JavaScript数据结构与算法》(第三版)，与第二版的区别在于加入了ES6和TypeScript。

刷算法前的准备

学习算法前需要对数据结构有一定的了解，懂得用JS实现基本的数据结构(栈、队列、链表等等)，不然你直接去力扣上刷肯定是从打开到关闭的。

完整的刷题路线建议到《代码随想录》 (opens new window)这个网站刷题，涵盖多个语言版本，从易到难、每个知识分模块的区刷题，比你无头苍蝇乱刷效率高太多了。

# 成果检验

Talk is cheap,show me the code.

纸上得来终觉浅,绝知此事要躬行。学习是需要输入+输出的，输入当然是你看文档、看视频，输出则是你笔记总结、做Demo练习、刷题做项目等等。 这里推荐几个网站来检验你的学习成果，

  • 牛客网 (opens new window)
  • 力扣 (opens new window)
  • 代码随想录 (opens new window)

首推肯定是牛客网专项练习，从选择题到手写编程题，以及各大大厂真题、面经等等，甚至力扣都不需要，这里额外提一嘴，力扣上的题目牛客基本都有，而且牛客的的手写题练习是ACM模式的，就是纯手敲的那种。力扣推荐《剑指offer》，代码随想录则不用说了，相见恨晚。

手写编程- 轮播图、tab页切换、表单验证、分页、节流防抖、深拷贝等等还有各大API的手写实现等等....

当然还有真实项目，但如果你是应届生，反而不那么重要，应届生的所谓项目并没有商业化的经验，在面试官眼里也只是玩具。项目本身并不重要，而是在项目中得到的锻炼，因此基础知识和手写能力(算法)足够优秀，项目经验就是可有可无的了，当然有肯定是加分项。其实算法在前端日常并不会接触，大厂考算法只是通过这个来筛人而已，属于无奈之举，工作了几年后考察项目经验比重会上升，考察算法比例会降低，抛开这些因素不谈，算法依然锻炼你的思维逻辑能力，值得重视。

项目可以是跟着视频一起做、可以是开源项目，建议找一个搭好的模板，看懂这个模板为什么这么设计，然后逐渐拓展加功能，项目可以选取博客、商城、后台管理系统这种，虽然烂大街，但是没做过嘛，依然能学习到许多东西。
