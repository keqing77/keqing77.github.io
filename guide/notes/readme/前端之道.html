# 前端前景

前后端都只是一个入门券


Stack Overflow 联合创始人 Jeff Wood 在2007年曾说过，任何一个能用 JavaScript 编写的应用系统，最终都必将使用 JavaScript 实现。这句话被称为阿特伍德定律(Atwood’s Law)

现在前端的火爆很大程度得益于node、HTML5、electron的出现，尤其是node，一大批后端人员涌进来，前端开始工程化，有了node 这个通用轮子后，可以用轮子造轮子，甚至可以跨平台造轮子，大量人才找到突破口，八仙过海各使神通，出现如今百家争鸣，前端工具大爆发的场面。H5抢了一部分IOS和安卓的饭碗，现在的前端不止是web端，可以踏足服务端领域以及移动端领域甚至桌面端、跨端,以及three.js为首的图形化和web游戏，现在非常火热的可视化和低代码平台搭建，前端人才缺口非常之大，内卷程度也不如早已红海的java。与其他互联网岗位相比，无论是测试、后端开发或是算法、运维、架构师，前端都是值得选择的性价比之选。

# 前端起步

前端开发工具选择


目前前端开发者工具使用率最高的是VSCode (opens new window)(75%~)，其次是Webstorm (opens new window)(20%~)、Sublime (opens new window)、Atom (opens new window)以及国产IDE HBuider (opens new window)。感兴趣可以到它们官网看一下。每个人的爱好都不一样，自己选择即可

个人推荐

# VsCode是世界上最好的Editor

如果你不知道怎么选择，那么我推荐你使用VSCode，微软开源的一个编辑器，功能强大界面简洁同时免费轻量，也是目前前端最为主流的开发工具，你几乎可以用VSC做一切事情，是真正的All in one。

# Webstorm是最强大但收费的Web IDE

如果你是JetBrain家的粉丝，那么WS不容错过，我个人也是VSC和WS配合使用，WS是真正的IDE(集成开发环境)，而VSC只是一个Editor(文本编辑器)，诸如路径跳转、项目管理...等等，ws是完爆vsc的，当然内存占用也完爆🤣，个人而言的话，大多数情况VSC够用，大型项目更喜WS，如果是免费的就更喜欢了。 当然使用atom、sublime、nodepad++甚至记事本都是没问题，如果你是vi/vim开发，请接收我的膜拜，Respect~

# 前端学习周期

WARNING

别听什么前端简单，前端，狗都不学😅

编程不是零门槛，更何况是编程开发。这些幼稚的言论和争论谁是最好的语言/框架一样非常的沙比，时代是变化的，前端现在的复杂程度是需要专业化的，但从历史原因来说，前端的确很长时间不受重视，如某些公司之前没有专门的前端岗位、学校不开展前端课程等等。实际上大多数的后端连浏览器开发者工具都不会使用，就更别提调样式、用JS写逻辑了。前后端不是对立的，本质都是程序员，只是不同领域的分工而已，网上这些言论只是为了秀优越感的无名之辈而已，一门语言/框架难=>这门语言/框架高级=>其他的low👎🏻，十分可笑。没有垃圾的语言，只有垃圾的开发者。

但编程对大多数人来说还不需要靠拼天赋，更多是你长时间坚持的努力练习。前端学习周期比起后端来说的确是要短，毕竟后端对算法和安全的要求比前端高得多，前端达到能干活水平，HTML5+CSS3+JavaScript是入场券，现在得加上一门现代JS框架(jq不算)。学习前端，肯定是越早越好，俗话说，种一棵树最好是十年前，其次是现在。详细学习路线和每个阶段花费时间的安排和需要的掌握情况可参考学习路线

程序员这个职业是终身学习的典范，无论是学习还是工作的时候都需要花费大量时间在学习提升技术上面。前端工程师要想成为架构师，就得吧前端这两个字拿掉，对不同领域都有所涉猎，不为自己的眼光设限。

# 程序员领域细分

这里介绍一些主要的程序员岗位和相对应掌握的技术，仅供参考，并不全面。

  • 前端工程师
    主要职责是web网页实现，除此之外现在的前端还涵盖移动端APP、小程序/公众号/H5/桌面端以及游戏可视化等领域， 需要掌握HTML+CSS+JS，以及可选Vue/React/Node/Flutter/Uni-app/Three.js...等等

  • 后端工程师
    相比于前端只有JavaScript，后端可选择的语言就五花八门了，(java，php，node，golang，python..)，主要是负责操作数据库和服务器端功能的实现(写接口)，需要掌握数据库和一门后端语言及相关技术栈。

  • 嵌入式开发工程师 非常偏硬件和软件底层，嵌入式的理解就是硬件里面嵌入程序，如洗衣机的控制芯片等，接触单片机和各种板卡芯片等。需要掌握集成电路相关知识，数字模拟电路，电路设计与实现、Linux、C/C++以及汇编等是必须掌握的。

  • 移动端开发工程师 web网页份额基本被前端抢去了，app也被抢走一定份额，前端火的原因也是因为抢了一部分移动端的饭碗。 不考虑前端做移动端的话，传统移动端分为IOS开发和安卓开发。 IOS开发: 必须拥有IOS设备，因为要用到Xcode，需要掌握Object-C、Swift。 安卓开发: 需要学习java、Linux等，还有新秀但是未来的Kotlin 其实和前端是满重合的，swift和js也是蛮像的，看swift真的有一种我会IOS的感觉。

  • 客户端/桌面端工程师 这个选择太多了，比较热门的有.Net平台的C#，以及Java，Python(QT)，甚至以electron为代表的前端er

  • 运维工程师 运维现在是朝着Devops方向，也就是开发部署，传统的运维是帮装软件的，掌控服务器，域名管理、性能监控、数据备份...等等， 需要掌握Linux、Nginx、shell、Python脚本编写，以及计算机原理、Docker、K8s...

  • 测试工程师 测试工程师是门槛最低的'程序员'了，甚至可以一行代码都不用敲，主要是找BUG写测试文档，非常的枯燥，工资也相对较低，但如果你是自动化脚本编写高手的话，那就另当别论了

  • 算法工程师 现实中我没见过也没接触过，但工资是这些岗位最高的(最卷的)

  • 游戏开发 国内基本是做手游， 需要掌握Unity、C/C++以及建模、图形学、游戏引擎等等(Directx/OpenGL)

  • 架构师 一个掌控整体又需要洞悉局部瓶颈并依据具体的业务场景给出解决方案的团队领导型人物，总之就是经验足够丰富、技术足够广且深的程序员。
