# 开发者应该具备的素质

TIP

授人以鱼不如授人以渔，教会你学这个东西不如教会你懂得如何学习。(提高自学能力和改善学习方法)

# 良好心态

焦虑学不会？没时间学？学习效率低？ 学不下去？遇到问题就畏惧？初学者有这心态是很正常，但是只要熬过去了就简单，遇到问题踩坑你应该开心，因为你踩过了坑并解决它这就锻炼了你的动手能力和解决问题能力，遇到困难也不会第一时间就抗拒，而是分析问题，心态很重要，编程并不需要天赋，只需要勤奋 + 动手练习。不会就是不会，没啥丢人的，学就完事了。打好基础才是最重要的。

# 打好基础

如果基础不牢的话，可能一个小问题就会卡你很久时间，需要花费非常多的时间去调试(debug),最终到最后解决问题才发现很多问题并没有想象中那么难，只是基础不扎实 或者⽂档没有仔细看。 基础是你技术的基⽯，⼀定要花时间打好基础，⽽不是追各种新的技术。⼀旦你的基础扎实，学习各种新的技术也肯定不在话下，因为新的技术，究其根本都是相通的。

# 抽时间学习

无论是在校还是已经工作，都需要抽时间去学习，学习是对自己最大的投资，坚持下去时间会给你最好的答案。

# 自学能力

没有自学能力的人是没有未来的

自学有两个因素需要深入考虑：未来的日子还很长，这世界进步得太快。 社会发展越来越快，我们要面对的人生越来越长，没有未来的日子，怎么过呢？自学能力是唯一值得被不断磨练的长技。 磨练出自学能力的好处在于，无论这世界需要我们学什么的时候，我们都可以主动去学，并且还是马上开始 —— 不需要等别人教、等别人带。 当然有很强的自学能力的意思也并不是说，什么都能马上学会、什么都能马上学好，到最后无所不精无所不通…… 因为这里有个时间问题。 无论学什么，都需要耗费时间和精力，与此同时更难的事情在于不断填补耐心以防它过早耗尽。 另外，在极端的情况下，多少也面临天分问题。 人学习就好像是动物进化一样…… 很多人很早就开始停止了进化。 在某些竞争中，先人一步意味着接受更多的教育、在健身房或图书馆里花更多的时间、获取更好的营养、付出更多的汗水、拥有更好的耐力或更好的设备。 这些都可以通过自学得到。

# 会看文档

初期学习可能你会习惯于看视频，但看视频时间花费太多了，而且年代不一定合适，而阅读文档的能力是每位开发者必备的。实际开发中，接触一门全新的技术边看文档边开发再正常不过了，而且最好是英文官方文档。中文文档更新不及时且翻译可能带来歧义。当然如果你是初学者，那么跟着视频做项目可能更为合适，毕竟官方文档质量也是参差不齐的，初学者基础薄弱看文档很可能抓不住重点，看不下去。但一定要学会看文档，仔细阅读⽂档⼀定会是少写 bug 的最省事路⼦。

# 学习英语

很多人可能认为自己英语差学不好英语，见到英文就抗拒，实际上英语没有这么难，也不是要你跟老外演讲，只需要掌握一些基本的词汇即可阅读。 我英文水平还凑合，但之前也是很抗拒的英文文档的，直到我硬着头皮去看，发觉我能看懂大部分，之后也不会感到困难，遇到不会的再去查就好了。程序员学习英语是非常简单的，因为我们在互联网⼀直接触着英语，并且看懂技术⽂章，⽂档所需要的单词量是极少的。很多初学者遇见报错可能会截图发群里问，但图里的报错英文提示非常清晰了，如果你的英语过关，完全不需要浪费时间来提问和搜索。学点英语绝对能带来能力和效率的提升。 学习英语方法太多了，chrome 装个翻译插件，直接拿英⽂⽂档或⽂章读，不会的就直接划词翻译，记录下这个单词并背诵。每天花半⼩时看点英⽂⽂档和⽂章，坚持两个⽉，看⽂档和⽂章绝对不会有难题了。这⼀定是⼀个很划算的个⼈时间投资， 花点时间学习英语，能为你将来的技术之路铺平很多坎。

# 懂得翻q

不会fq说明你不是一名合格的开发者，会不会是一个筛子，筛掉没有独立思考能力的人在墙内。如果你真的不会，请你仔细思考一下该用什么搜索引擎搜什么关键字或者去哪里找，总之这个你得会，我会也不能教，毕竟传播违法。至于fq后干什么，那当然是学习啊！难道你想看P站？

# 懂得搜索

搜索引擎最大的作用就是输入问题获得答案，但是百度，我就不想说话了，我有1万个理由证明百度是个垃圾搜索引擎。(漫天广告、竞价排名..)。同样⼀个关键字，使⽤ 百度和⾕歌，⾕歌基本完胜的。即使你使⽤中⽂在⾕歌中搜索，得到的结果也往往是谷歌最能为你呈现你想要的答案，所以如果你想迅速的通过搜索引擎来解决问题，那⼀定是⾕歌/必应。

# 懂得总结

学习是需要输入输出的，学了不一定会，看了不敲=白学。强烈建议学习前端经营自己的个人博客、公众号。。，不仅面试加分，而且 重要的是培养平时归纳总结的习惯。学习最快的方法就是教会别人，别人越小白你越能把这个问题讲清楚，说明你是真正理解了。 再强烈推荐一款好用的笔记软件，notion。具体原因可以看我掘金发布的文章

# 懂得思考

需求出来不要立马闷头做，毕竟需求会随时更改而且出问题返工是很正常的。拿到设计稿和需求先花点时间想一想，画一画，看看之前的组件是否可以复用，这个界面是怎么组织的，数据之间的传递是这样的。先画一个流程图，再动手做也许是更好的方法。
