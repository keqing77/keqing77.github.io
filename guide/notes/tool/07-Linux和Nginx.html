# Linux目录和文件颜色含义

可以使用tree查看项目结构

THEOREM

常用目录的作用如下： bin： 存放普通用户可执行的指令，普通用户也可以执行； dev ： 设备目录，所有的硬件设备及周边均放置在这个设备目录中； boot ： 开机引导目录，包括 Linux 内核文件与开机所需要的文件； home： 这里主要存放你的个人数据，具体每个用户的设置文件，用户的桌面文件夹，还有用户的数据都放在这里。每个用户都有自己的用户目录，位置为：/home/用户名。当然，root 用户除外； usr： 应用程序放置目录； lib： 开机时常用的动态链接库，bin 及 sbin 指令也会调用对应的 lib 库； tmp： 临时文件存放目录 ； etc： 各种配置文件目录，大部分配置属性均存放在这里；

TIP

蓝色 表示文件夹； 灰色 表示普通文件； 绿色 表示可执行文件； 红色 表示压缩文件； 天蓝色 表示链接文件（快捷方式

# Nginx

Nginx功能

  • 图片防盗链
  • 反向代理跨域
  • 负载均衡
  • 请求过滤
  • 动静分离

THEOREM

  • /etc/nginx/nginx.conf 主配置文件会默认把这个文件夹中所有子配置项都引入
  • /etc/nginx/conf.d/ 文件夹，是我们进行子配置的配置项存放处，
  • /usr/share/nginx/html/ 文件夹，通常静态文件都放在这个文件夹，也可以根据你自己的习惯放其他地方；
# 安装niginx
yum list | grep nginx
yum install nginx
nginx -v

#启动 Nginx 
systemctl start nginx

# Nginx开机自启
systemctl enable nginx


# 重启nginx
nginx -s reload 

# 配置Nginx
vim /etc/nginx/nginx.conf

# Nginx反向代理解决跨域
location /api {
	proxy_pass 跨域地址的baseURL;
}

# 开启Gzip压缩
cd /etc/nginx/conf.d/
touch gzip.conf
vim gzip.con
gzip on; # 默认off，是否开启gzip
gzip_types text/plain text/css application/json application/x-javascript text/xml application/xml application/xml+rss text/javascript;
