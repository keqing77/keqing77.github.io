# 什么是git？

git是一个开源免费的分布式版本控制系统，由Linux之父Linus Torvalds用C语言在花费一周时间编写的一款应用。从名字就可以看出它的特点

  • 开源、免费
  • 分布式
  • 版本控制
  • 系统
  1. 开源、免费。开源-源代码公开，你可以自己修改。免费-不用担心收费，方向用。正因如此，git是现在最流行的版本控制软件，每一位开发者都需要掌握git。
  2. 分布式的意思就是没有服务端跟客户端之分，只要你安装了git，那么你就可以拉取其他人的仓库代码，同样地别人也可以拉取你的代码。Github是全球最大的基于Git的代码托管网站。
  3. 版本控制的意思就是字面意思，控制不同的版本发布，实际项目都是多人协同开发，有了git就可以各自开发，最后合并并且控制发布版本，提升了开发速率。
  4. 系统，git是一个完整的分布式系统，也就是说一个系统该有的功能它都有。其中最后核心的是分支，使用分支意味着你可以把你的工作从开发主线上分离开来,以免影响开发主线，同时基于不同的分支可以控制不同的版本迭代，方便版本回滚等。

# 学习git

WARNING

Git并不简单

git是一个完整的分布式系统，代码冲突、分支合并、撤销提交、远程追踪、分离Head等等。真正掌握git是得需要团队协作中掌握，git的使用是需要相当谨慎的，不然一不小心就会造成冲突酿成大祸。我实习连git都不会，需求不断来，需要切换不同分支工作，遇到各种冲突更是令人头大，稍不注意就被坑半天，归根结底在于对git的掌握不够深，务必引起对Git的重视，如果不会使用Git，那么会给其他人非常差的印象😭。如果你使用jetbrain家的IDE有很好的git可视化工具，如果是Vscode，那么推荐git lens和git graph~~~

# 配置Git命令别名

实际开发中可能图形化工具会更直观，如vscode插件、webstorm、sourcetree等等，毕竟命令行比较简陋，但最基本的命令肯定是滚瓜烂熟的。但是一遍遍敲重复的命令实在没意思，所以你可以配置alias别名来缩写，要配置在你的终端文件上，如果你使用的是bash则配bash配置文件，我使用的是zsh，所以配置.zshrc文件。

alias gc='git clone'
alias gs='git status'
alias gd='git diff'
alias ga='git add .'
alias gcm='git commit -m'
alias gb='git branch'
alias gba='git branch -a'
alias gp='git pull'
alias gpu='git push'
alias gl='git log'

# Git常用命令

git rebase
git status
git diff
git merge
git checkout 
git switch
git reset 
git Branch
git fetch


git add .                                将所有改动放进暂存区
git commit -m "描述"                      提交并附带概要信息
git pull                                 从远程仓库拉去代码
git push                                 推送代码到远程仓库（master分支）

git log                                  查看日志
git log -p                               查看详细历史
git log --stat                           查看简要统计
git log --oneline --graph 		 查看简单历史，显示树
git reflog                               查看操作记录

git branch 名称                          创建分支
git checkout 名称                        切换分支
git checkout -b 名称                     创建并切换到新分支
git checkout --track 名称                创建新分支并关联新分支
git branch -d 名称                       删除该分支（不能删除当前所在的分支，不能删除没有合并到master上的分支）
git branch -D 名称                       删除该分支（可以删除没有合并到master上的分支）

git commit --amend                      对最新的一条commit进行修正
git reset --hard HEAD^                  丢弃最新提交（未提交的内容会被擦掉）
git reset --soft HEAD^                  丢弃最新提交（未提交的内容不会被擦掉）
git revert HEAD^                        回到某个commit
git rebase 目标基础点                     重新设置基础点
git pull --rebase 			从远程仓库拉去代码并重新设置基础点
git merge 分支名称                       将分支合并到当前分支
git push origin branchName              将代码推送到远程仓库的指定分支
git push -d origin branchName           删除远程分支

git stash                               暂存代码(直接保存当前修改的文件代码)
git stash list                          查看暂存代码列表
git stash apply                         命令时可以通过名字指定使用哪个stash，默认使用最近的stash（即stash@{0}）
git stash pop                           弹出暂存代码（出栈，删除list的这行记录）
git stash drop                          删除指定的stash记录
