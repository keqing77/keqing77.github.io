    # 事件循环

    WARNING 这里的事件循环是在浏览器环境下的，并非node环境下，node事件循环单独放在node篇

    # 进程和线程

    首先JavaScript一门单线程语言，什么是单线程？就是一次只能做一件事？人类就是多线程，比如一边思考🤔一边拉屎，能同时干好几件事。为什么JS要被设计成单线程呢？你试想，如果一个DOM被几个JS函数操作该听谁的？加上线程间切换的开销，无疑会让开发者混乱。

    先简单说下进程和线程，就拿浏览器冲浪举例子，你打开一个标签页实际上就是开启了一个进程，一个进程中又可以有多个线程，如JavaScript引擎线程和渲染页面的线程。这两个线程是互斥的，也就是说当JavaScript在执行的时候，渲染进程是被阻塞的，反之同理。

    # 执行栈

    执行栈就是一个执行JS事件的栈，遵循先进后出的理念。

    JS引擎遇到异步的代码时，会被挂起并在需要执行的时候加入到 Task（有多种 Task）队列中。一旦执行栈为空，Event Loop 就会从 Task 队列中拿出需要执行的代码并放入执行栈中执行。Task又被分为宏任务(macrotask 称为 task)和微任务(microtask 称为 jobs)

    1，同一个执行上下文中同步任务优先于异步任务

    2，不同执行环境中的异步任务执行先后取决于其加入到事件队列的时间先后

    总的结论就是，执行宏任务，然后执行该宏任务产生的微任务，若微任务在执行过程中产生了新的微任务，则继续执行微任务，微任务执行完毕后，再回到宏任务中进行下一轮循环。

    答案

    这里需要先理解async/await。

    async/await 在底层转换成了 promise 和 then 回调函数。 也就是说，这是 promise 的语法糖。 每次我们使用 await, 解释器都创建一个 promise 对象，然后把剩下的 async 函数中的操作放到 then 回调函数中。async/await 的实现，离不开 Promise。从字面意思来理解，async 是“异步”的简写，而 await 是 async wait 的简写可以认为是等待异步方法执行完成。

    await下面的代码 = promise.then() 可以把 await 看成是让出线程的标志

    async function async1() {
  console.log("async1 start");
  await async2();
  console.log("async1 end");
  setTimeout(() => {
    console.log('timer1')
  }, 0)
}
async function async2() {
  setTimeout(() => {
    console.log('timer2')
  }, 0)
  console.log("async2");
}
async1();
setTimeout(() => {
  console.log('timer3')
}, 0)
console.log("start")
