    # this的指向

    this的指向绝对是学习JavaScript最令人困扰的问题之一，实际不难理解。

    this的绑定规则

    其实一句话就可以概括: 谁调用this，this指向谁。

    • 直接调用fn()，不管 fn 函数被放在了什么地方，this 一定是 window
    • obj.fn() ，我们只需要记住，谁调用了函数，this 就指向谁
    • new对象，this 被永远绑定在了new创建的对象 上面，不会被任何方式改变 this

    # this的四种绑定规则( 1、默认绑定 2、隐式绑定 3、显式绑定 4、new绑定)

    1. 默认绑定
    2. 隐式绑定
    3. 显式绑定
    4. new绑定

    当然还有箭头函数和严格模式 这两种情况

    1.箭头函数没有自己的this，会找第一个包含自己的非箭头函数的this当做自己的this

    2.严格模式下，在函数内部的this禁止指向全局对象，值为undefined。
    如果你不了解严格模式，那么可以看严格模式倒地严格在哪里

    this4种绑定的优先级 new绑定 > 显式绑定 > 隐式绑定 > 默认绑定

    也就是说当1个this拥有不同规则，会按照以上优先级绑定this，其中最强的是new绑定，通过new的this绑定无法改变，永远都是指向创建的对象，不会被任何方式改变(包括call;applu;bind)最弱是默认绑定，其他规则不存在才会运用。

    当然也可以一句话归纳： 谁调用this，this指向谁。

    普通函数这里的this的指向是不明确的，直到调用的时候才知道，而箭头函数的this的指向是在定义的时候就已经确定了。(因为没有自己的this)

    总结: new 绑定this的方式优先级最高，接下来是 bind 这些函数，然后是 obj.fn() 这种调用方式，最后是 fn() 这种调用方式，

    不会被改变的this

    箭头函数的 this 一旦被绑定，就不会再被任何方式所改变

    多次bind绑定，this也只会取第一次bind的this

    # 1.默认绑定

    形式: fn() 无论fn()在什么位置，都是默认绑定指向全局对象(浏览器下是window node下是global)

    谁调用this，this指向谁。实际上可看做 window.fn()

    function fn(){
    console.log(this.a); // 2

    # 3.显式绑定

    形式: fn.call(xxx1,xxx2) apply bind 同理

    • call apply bind 三者都是为了改变this的指向
    • 不同点在于call接收一个参数列表,apply接受一个参数数据，bind则是返回一个函数对象
    • 还有区别就是 bind()是强绑定，之后无论怎么调用这个函数，this的指向都不会被改变。而call和apply是在函数在被调用的时候临时改变this的指向，不影响之后的行为。

    显式绑定最容易分辨了，因为this指向谁已经写出来了

    1.最常见的就是通过call,bind,apply方法改变this的指向使其绑定的参数所指的对象

    (this 取决于第一个参数，如果第一个参数为空，那么就是 window)

    2.在使用数组的forEach/map/filter/some/every等方法时，也可以传入对象，显式绑定this

    const steven = {
    name:'Steven',
    phoneBattery:70,
    charge:function(level) {
        this.phoneBattery = level
    }
}

const becky = {
    name:'Becky',
    phoneBattery:30,
}

console.log(steven);
steven.charge(100);
console.log(steven);

steven.charge.call(becky, 95);
console.log(becky);

    # 4.new绑定

    new绑定将this绑定到创建的对象上，并且不会被任何方式改变this！！！ (new绑定优先级最高，即通过call apply也改变不了)

    new所作的四件事情：（摘自MDN）

    1. 创建一个空的简单JavaScript对象（即**{}**）；
    2. 为步骤1新创建的对象添加属性，将该属性链接至构造函数的原型对象 ；proto
    3. 将步骤1新创建的对象作为**this**的上下文 ；
    4. 如果该函数没有返回对象，则返回**this**。

    # 5.箭头函数

    箭头函数表达式的语法比函数表达式 (opens new window)更简洁

    箭头函数没有自己的this

    箭头函数会去找上层常规函数(Function)的this作为自己的this，如果找不到就将this指向全局对象。（简单来说就是this指向包含箭头函数的常规函数，如果没有常规函数的话就是全局对象）

    同时，箭头函数的 this 一旦被绑定，就不会再被任何方式所改变

  radius: 10,

  perimeter: () => 2 * Math.PI * this.radius
}
