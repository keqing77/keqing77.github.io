# 什么是继承?

继承就是子类可以使用父类的所有功能，且可以对这些功能进行扩展。但是要知道，在JS中是不存在’类’的， JS中的类 本质上是 对象 委托 另一个对象 (函数也是对象) ，很多概念你都不会用传统面向对象语言(Java,C#等) 去套用JavaScript的概念去理解。

彻底理解继承前需要了解一下概念,非常重要！

# 传统的类

在传统的面向对象语言中，如(Java,C#,C++),都存在类（class）的概念，类就是对象的模板，对象就是类的实例。是在Javascript语言体系中，是不存在类（Class）的概念的，javascript中不是基于‘类的'，而是通过构造函数（constructor）和原型链（prototype chains）实现的。虽然ES6中提供了更接近传统语言的写法，引入了Class（类）这个概念，作为对象的模板。通过class关键字，可以定义类。但ES6的class只是一个语法糖而已，新的class写法只是让原型对象的写法更加清晰、更像面向对象编程的语法而已。

# 需要先有的概念

  • 实例方法只有实例可以调用
  • 静态方法只有构造函数可以调用
  • 原型方法是实例和构造函数都可以调用，是共享的方法。

构造函数

在 JavaScript 中，构造器其实就是一个普通的函数。当使用 new 操作符 来作用这个函数时，它就可以被称为构造方法（构造函数

// 1. 构造函数   
function Person (name) {this.name}

// 2. 构造函数的原型对象
Person.prototype

// 3. Person构造函数创建的实例对象 '张三'
let 张三 = new Person('张三')

// 4. 构造函数的静态方法,名为fn
Person.fn = function () {}

// 5. 原型对象上的方法,名为fn 
Person.prototype.fn = function () {}

# 1.原型继承---实现

// 原型链继承
function Foo() {
    this.a = "111";
  }
  
  Foo.prototype.say = function () {
    console.log("这是一个方法");
  };
  
  function Bar() {}
  
  Bar.prototype = new Foo(); //将原型对象指向别的实例  就可以继承实例原型链上的方法
  
  var bar1 = new Bar();
  
  console.log(bar1.a);
  // Bar并没有属性a，但是Bar.prototype.a = 111，所以bar1.a = 111
  bar1.say();
  // bar是Bar new出来的实例，但是Bar上没有方法，而Bar.prototype上有方法，所以bar.say()调用的是Bar.prototype.say()

//原型链的缺点在于:会有引用值共享的问题，修改1个实例的引用值会影响另一个实例

function Foo() {
  // this.a = '111'; 原始值没问题
  this.a = [1, 2, 3, 4];
}

Foo.prototype.say = function () {
  console.log("这是一个方法");
};

function Bar() {}

Bar.prototype = new Foo();

var bar1 = new Bar();
var bar2 = new Bar();

bar1.a.push(5);
console.log(bar1.a); //[ 1, 2, 3, 4, 5 ]
console.log(bar2.a); //[ 1, 2, 3, 4, 5 ]
//bar1修改引用值 影响了bar2

# 2.构造函数继承---实现

function Foo() {
  // this.a = '111'; 原始值没问题
  this.a = [1, 2, 3, 4];
}

Foo.prototype.say = function () {
  console.log("这是一个方法");
};

//在子类构造函数内部使用call或apply来调用父类构造函数
function Bar() {      //构造函数继承
  Foo.call(this);
}

// Bar.prototype = new Foo();   这是上面的原型继承 

var bar1 = new Bar();
var bar2 = new Bar();

bar1.a.push(5);  //修改引用类型也互不影响
console.log(bar1.a); //[ 1, 2, 3, 4, 5 ]
console.log(bar2.a); //[ 1, 2, 3, 4 ]

// 构造函数继承问题: 没办法拿到原型上的方法

# 3.组合继承---实现

  function Foo() {
    this.a = [1, 2, 3, 4];
  }
  
  Foo.prototype.say = function () {
    console.log("这是一个方法");
  };
  
  function Bar() {
    Foo.call(this);
  } //构造函数继承
  
  Bar.prototype = new Foo(); //原型继承
  
  //两者组合就是组合继承  但是Foo()被执行了两次
  
  var bar1 = new Bar();
  var bar2 = new Bar();
  
  bar1.a.push(5);
  console.log(bar1.a); //[ 1, 2, 3, 4, 5 ]
  console.log(bar2.a); //[ 1, 2, 3, 4 ]

# 4.寄生组合继承---实现

function Foo() {
  this.a = [1, 2, 3, 4];
}

Foo.prototype.say = function () {
  console.log("这是一个方法");
};

function Bar() {
  Foo.call(this);
} //构造函数继承

//Bar.prototype = new Foo(); //原型继承，结合起来是组合继承，Foo()被执行了两次

Bar.prototype = Object.create(Foo.prototype);
//通过Object.create()方法直接指定原型对象，不需要new构造函数再次调用Foo()

var bar1 = new Bar();
var bar2 = new Bar();

bar1.a.push(5);
console.log(bar1.a); //[ 1, 2, 3, 4, 5 ]
console.log(bar2.a); //[ 1, 2, 3, 4 ]

# 5.class继承---实现

class Parent {
  constructor(value) {
    this.val = value
  }
  getValue() {
    console.log(this.val)
  }
}
class Child extends Parent {
  constructor(value) {
    super(value)
    this.val = value
  }
}
let child = new Child(1)
child.getValue() // 1
child instanceof Parent // true

# 6.圣杯继承---实现

构造函数

构造函数是用new创建对象时调用的函数，与普通唯一的区别是构造函数名是首字母大写 (只是这么规定，不是必须)

为什么要new呢？ 是让this 指向创建的对象，不然是全局对象window。

constructor

每个对象在创建时都自动拥有一个构造函数属性constructor，其中包含了一个指向其构造函数的引用。而这个constructor属性实际上继承自原型对象，而constructor也是原型对象唯一的自有属性

原型对象及prototype

通过构造函数的new操作创建实例对象后，会自动为构造函数创建prototype属性，该属性指向实例对象的原型对象。通过同一个构造函数实例化的多个对象具有相同的原型对象。下面的例子中，Foo.prototype是原型对象

function Foo(){}
Foo.prototype.a = 1
var f1 = new Foo
var f2 = new Foo 
console.log(Foo.prototype.a)
console.log(f1,a) 
console.log(f2.a)

 isPrototypeOf()
可以通过isPrototypeOf()方法来确定对象之间是否是实例对象和原型对象的关系
Object.getPrototypeOf()

ES5新增了Object.getPrototypeOf()方法，该方法返回实例对象对应的原型对象

属性查找

当读取一个对象的属性时，javascript引擎首先在该对象的自有属性中查找属性名字。如果找到则返回。如果自有属性不包含该名字，则javascript会搜索proto中的对象。如果找到则返回。如果找不到，则返回undefined

in

in操作符可以判断属性在不在该对象上，但无法区别自有还是继承属性

hasOwnProperty()

通过hasOwnProperty()方法可以确定该属性是自有属性还是继承属性
