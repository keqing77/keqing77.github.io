# 变量和数据类型

# 数据类型

JS数据类型分为: 基本数据类型(七种)和引用类型(一种)，共八种类型

1.基本数据类型(7种): 原始值

string number boolean null undefined symbol(es6) bigint(es10)
1

其中symbol为es6、bigint 为es10后新增的数据类型

  • symbol表示独一无二的值，,且不可枚举的，object.keys无法列举 ,因此可以作为对象的私有属性等，要找同一个symbol可以用symbol.for方法。
  • bigint表示表示任意大的整数，可以表示大于 2^53-1 的整数
💡 JS 的数字类型是浮点类型的，没有整型。并且浮点类型基于 IEEE 754标准实现（64位），基于该标准的语言都存在浮点数运算精度丢失的问题(PHP,Java，Python等)。

2.引用数据类型(1种):

object   [function array Date Regexp....]
1

基本数据类型与引用类型不同的是，它们存放在栈中(stack)，大小在创建的时候就固定了，存储的是值； 而引用类型是存放在堆中(heap)，大小不固定，在栈中存储的是指向该对象的指针！

所以使用引用类型会存在深浅拷贝的问题，基本数据类型是不存在深浅拷贝问题的！

NaN 是属于 number 类型，以及NaN判断相等

Object.js(NaN,NaN) // true

以及 NaN === NaN //false

堆和栈

  • 栈 自动分配大小固定的内存空间，自动释放，遵循先进后出原则,存放基本数据类型以及指向存放在堆中引用类型的地址指针。
  • 堆 大小不固定，存放引用类型，保持引用则垃圾回收不会回收。

基本类型采用的是值传递，而引用类型是地址传递。结合下图理解，就可以知道，如果找一个基本类型是直接去栈中找，而找一个引用类型，则会先去找它的地址指针(存放在栈中)，然后通过这个地址找到在堆内存的位置从而找到引用类型。

# 深浅拷贝

概念：浅拷贝只复制指向某个对象的指针，而不复制对象本身，新旧对象还是共享同一块内存。但深拷贝会另外创造一个一模一样的对象，源对象与拷贝对象互相独立，新对象跟原对象不共享内存，修改新对象不会改变旧对象。

深拷贝和浅拷贝都是针对引用类型数据，因为基本数据类型赋值后（开辟新内存空间存放新值），改变新数据，都不会影响到原来的数据。也就是说对于基本数据类型来说，无论深浅拷贝，数据都是共用的，只要一个拷贝的值发生改变，另外一个不会受影响 。

浅拷贝： 直接赋值、 object.assign、 concat() 、slice()、 Array.from() ...拓展运算符 ，以上都是浅拷贝，除了直接赋值，后面的方法如果只涉及基本数据类型和单层引用类型，你也可以认为他们是(伪)深拷贝。

看到这可能开始抗拒了，这么多？记不住哇！其实你只要记住不是深拷贝(就2种)都是浅拷贝就得了~

深拷贝：

  1. JSON.parse(JSON.stringify(object))
  2. 手写深拷贝

JSON这种方法虽然快速，但有几个缺点。如

  • 会忽略 undefined
  • 会忽略 symbol
  • 不能序列化函数
  • 不能解决循环引用的对象

更好的处理方式当然是手写深拷贝了，但手写深拷贝需要处理许多问题，很多人写的其实都没有考虑周到，比如原型链如何处理、DOM 如何处理等等，推荐使用lodash (opens new window)的深拷贝函数。

# 浮点数精度丢失

前面的图已经展示了，数字的位数只有53位，而一些小数用二进制表示是无限循环的，只能截取这么多位来表示，所以它们计算的结果会有偏差，这就是浮点数精度丢失的问题。





 

console.log(0.1 + 0.2 = 0.30000000000000004)
0.1 + 0.2 != 0.3  // true

//如何解决？
parseFloat((0.1+0.2).toFixed(10)) ===0.3
