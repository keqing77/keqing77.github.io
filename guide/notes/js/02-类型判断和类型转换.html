# 类型判断和类型转换

# JS判断数据类型的方法

1. typeof-----基本数据类型类型
2. instanceof-----引用类型
3. Object.prototype.toString().call() -----所有类型
4. Array.isArray()、Object.is()、 isNaN() Number.isNaN()...etc-----特殊类型判断
5. constructor-----看是谁构造的

# typeof

用法： typeof 变量名

返回值 ：表示该变量类型名的小写字符串

  • 可以判断基本数据类型，除了Null (null是object，这是1个设计上的BUG)
  • 不能判断引用类型，因为都统一返回'object'
typeof 666 // 'number'
typeof '0' // 'string'
typeof undefined // 'undefined'
typeof true // 'boolean'

const sym1 = Symbol();
typeof sym1 // 'symbol'

const HugeNumber = BigInt(9007199254740991);
typeof HugeNumber //"bigint"

typeof null //'object' 基本数据类型除了null都是判断准确的。这情况是JS设计的一个缺陷
typeof NaN //'number' NaN = not a number 所以NaN是Number类型，JS真有你的哦~

typeof并不能准确判断引用类型，因为除了函数外返回的都是'object'
typeof function(){}  //'function'
typeof [1,2,3] //'object'

# instanceof

用法：A instanceof B

返回值 ：布尔值，true 或false

  • 可以判断引用类型判断引用类型,检测构造函数的 prototype 属性是否出现在该实例对象的原型链上
// 定义构造函数
function C(){}
function D(){}

var o = new C();

o instanceof C; // true，因为 Object.getPrototypeOf(o) === C.prototype
o instanceof D; // false，因为 D.prototype 不在 o 的原型链上
o instanceof Object; // true，因为 Object.prototype.isPrototypeOf(o) 返回 true
C.prototype instanceof Object // true，同上

C.prototype = {};
var o2 = new C();

o2 instanceof C; // true
o instanceof C; // false，C.prototype 指向了一个空对象,这个空对象不在 o 的原型链上.

D.prototype = new C(); // 继承
var o3 = new D();
o3 instanceof D; // true
o3 instanceof C; // true 因为 C.prototype 现在在 o3 的原型链上

# Object.prototype.toString().call()

用法：Object.prototype.toString().call(变量名)

返回值 ：返回一个表示该对象的字符串

这是判断类型最为准确的方法，能够准确判断基本数据类型(包括null)和引用类型的方法

  • 为什么不直接toString(XX)呢？

toString() 方法返回一个表示该对象的字符串，call()让this指向该对象。

每一个继承 Object 的对象都有 toString 方法,对于大多数对象(Array、Date、RegExp等)继承自Object的，toString() 方法都是重写了的，这时就需要用 call() 或 Reflect.apply() 等方法来调用。

我们可以直接调用Object原型上未被覆盖的toString()方法，使用call来改变this指向来达到我们想要的效果。

// Boolean 类型，tag 为 "Boolean"
Object.prototype.toString.call(true);            // => "[object Boolean]"

// Number 类型，tag 为 "Number"
Object.prototype.toString.call(1);               // => "[object Boolean]"

// String 类型，tag 为 "String"
Object.prototype.toString.call("");              // => "[object String]"

// Array 类型，tag 为 "String"
Object.prototype.toString.call([]);              // => "[object Array]"

// Arguments 类型，tag 为 "Arguments"
Object.prototype.toString.call((function() {
  return arguments;
})());                                           // => "[object Arguments]"

// Function 类型， tag 为 "Function"
Object.prototype.toString.call(function(){});    // => "[object Function]"

// Error 类型（包含子类型），tag 为 "Error"
Object.prototype.toString.call(new Error());     // => "[object Error]"

// RegExp 类型，tag 为 "RegExp"
Object.prototype.toString.call(/\d+/);           // => "[object RegExp]"

// Date 类型，tag 为 "Date"
Object.prototype.toString.call(new Date());      // => "[object Date]"

// 其他类型，tag 为 "Object"
Object.prototype.toString.call(new class {});    // => "[object Object]"

# 特殊类型的判断方法

WARNING

判断数组 Array.isArray()
判断对象 Object.is()
判断NaN isNaN()、Number.isNaN()

Array.isArray([1, 2, 3]);  // true  判断是否是数组

Object.is(NaN,NaN);  //true Object.is() 方法判断两个值是否为同一个值

isNaN("37")  //false: 可以被转换成数值37  如果isNaN函数的参数不是Number类型， 
//isNaN函数会首先尝试将这个参数转换为数值，然后才会对转换后的结果是否是NaN进行判断
//因此并不准确，判断NaN更推荐采用Number.isNaN()

// 下面这几个如果使用全局的 isNaN() 时，会返回 true。
Number.isNaN("NaN");      // false，字符串 "NaN" 不会被隐式转换成数字 NaN。
Number.isNaN(undefined);  // false
Number.isNaN({});         // false
Number.isNaN("blabla");   // false

# == 、===的区别，

  • ==只比较值的相等，===会比较值和类型的相等，
  • 当两边类型相等的时候，==和===的结果是一样的，

# ===和Object.is()的区别

  • 二者区别只有判断NaN和带符号的0，其他结果都是一样的
NaN === NaN // false
+0 === -0 // true
-0 === +0 // true

Object.is(NaN, NaN) // true
Object.is(+0, -0) // false
Object.is(-0, +0) // false

# ==判断的流程是怎样的？

这涉及JS的一个重难点 ,隐式类型转换！

对于==来说，判断相等会经过以下步骤

  1. 两者类型是否相等，相等则比较值的大小
  2. 类型不相等，则进行类型转换
  3. 判断类型是否是 null 和 undefined，是的话就会返回 true
  4. 判断两者类型是否为 string 和 number，是的话就会将字符串转换为 number
  5. 判断其中一方是否为 boolean，是的话就会把 boolean 转为 number 再进行判断
  6. 判断其中一方是否为 object 且另一方为 string、number 或者 symbol，是的话就会把 object 转为原始类型再进行判断

# 逻辑判断(!! && ||)和比较运算符 (> <)

字符串的比较

  • 字符串的比较是比较ASCII码，而不是字符串的内容
  • 逐位进行比较，相等则比较下一位
  1. 其中数字0~9对应的ASCII码(十进制)是48~57
  2. 大写字母A~Z对应的ASCII码(十进制)是65~90
  3. 小写字母a~z对应的ASCII码(十进制)是97~122
null == 0 // false

null > 0 // false

null < 0 // false

null >= 0 // true

null <= 0 // true

[undefined] == false // true

undefined == false // false

[] == ![] // true

[] == [] // false

{} == {} // false

{} == !{} // false

[] == 0 // true

[2] == 2 // true

['0'] == false // true

'0' == false // true

[] == false // true

[null] == 0 // true
