# readonly只读属性

响应式对象的状态是变化的，你不知道是谁修改了什么，因为状态可用并且可以直接更改，没有加以限制。

这时候你就可以使用readonly函数将状态包装起来，来表示响应式对象不可修改，它覆盖了在proxy对象中传递的变量，该代理对象阻止任何修改(在尝试修改的时候发出警告)。这些变化可以由访问可写存储的单独函数来处理。

import {reactive,readonly} from 'vue';

export const createStore = () => {
	const state = reactive({counter:0});
  const increment = () => state.counter++;

return {increment,state:readonly(state)}
}

外部将只能访问只读状态，并且只有导出的函数可以修改可写状态。

<template>
    <h1>Vue3新特性 - readonly只读属性</h1>
	<ul>
		<li>{{nameObj.name}}</li>
	</ul>
</template>


<script>
    import { reactive, readonly } from 'vue'

    export default {
        setup()
        {
            let nameObj = reactive({name: '张三'})
            let readonlyObj = readonly(nameObj) 
						// 对nameObj响应式对象设置成只读,不可修改
            setTimeout(() => {
 // 无法设置属性=> et operation on key "name" failed: target is readonly. Proxy {name: "张三"}
                readonlyObj.name = 'zhangsan'
            }, 1000)

            return {
                nameObj
            }
        }
    }
</script>
