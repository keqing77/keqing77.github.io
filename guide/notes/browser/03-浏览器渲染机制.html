# 浏览器渲染机制

提前需要知道的概念

  • DOM：Document Object Model，浏览器将HTML解析成树形的数据结构，简称DOM。
  • CSSOM：CSS Object Model，浏览器将CSS解析成树形的数据结构，简称CSSOM。
  • Render Tree: DOM和CSSOM合并后生成Render Tree
  • css加载不会阻塞DOM树解析，但是会阻塞DOM树渲染
  • css加载会阻塞后面js语句的执行
  • JS 的加载、解析与执行会阻塞DOM 的构建

经典面试题

从输入url到浏览器显示页面发生了什么？

答案
  • 输入url发起http请求，然后开始进行DNS解析（优化：预解析）
  • 建立TCP连接(三次握手，四次挥手，websocket等等)，
  • 建立起http、https连接，（https还要经过SSL/TLS 的一层加密），
  • 得到服务器的响应(响应头、状态码 200 304 404之类)
  • 浏览器得到资源文件，开始解析，解析html生成dom，
  • 遇到link加载CSS同时继续解析（CSS的加载不会影响DOM的解析），遇到script标签需要等待js文件加载完成，js是会阻塞DOM的解析的（所以需要把js放在页面最底- 部），请求JS和CSS返回的顺序是未知的，就算先返回并且完成解析JS文件也是会发生阻塞，
  • 必须等到CSSOM构建完成后才能执行JS文件。因为JS是可以控制DOM和CSS样式的！！！
  • 可以在script标签内添加defer和async进行异步解析，就不会阻塞DOM的解析，js的解析加载背后的原理有(v8引擎解析运行，AST->机器码->字节码，事件循环、垃圾回收和内存管理等等。)
  • DOM加载完成后还是要等CSSOM完成后才会渲染，生成render树 也就是页面开始渲染
  • 调用 GPU 绘制，合成图层，显示在屏幕上(重排、重绘)

# 重排(回流)与重绘

Reflow重排(回流)


当Render Tree中部分或全部元素的尺寸、结构、或某些属性发生改变时，浏览器重新渲染部分或全部文档的过程称为回流

Repaint重绘


当页面中元素样式的改变并不影响它在文档流中的位置时（例如：color、background-color、visibility等），浏览器会将新样式赋予给元素并重新绘制它，浏览器重新绘制元素的过程称为重绘

会导致回流的操作：

  • 页面首次渲染
  • 浏览器窗口大小发生改变
  • 元素尺寸或位置发生改变
  • 元素内容变化（文字数量或图片大小等等）
  • 元素字体大小变化
  • 添加或者删除可见的DOM元素
  • 激活CSS伪类（例如：:hover）
  • 查询某些属性或调用某些方法

一些常用且会导致回流的属性和方法：

clientWidth()、clientHeight()、clientTop()、clientLeft()
offsetWidth()、offsetHeight()、offsetTop()、offsetLeft()
scrollWidth()、scrollHeight()、scrollTop()、scrollLeft()
scrollIntoView()、scrollIntoViewIfNeeded()
getComputedStyle()
getBoundingClientRect()
scrollTo()

# defer和async区别

我们知道，浏览器解析HTML是一行一行按顺序往下面执行的，当浏览器读取到<script>的时候，便会暂停解析DOM，而是like开下载JS脚本，并在下载后立即执行，这个特性会导致DOM还没构建完页面白屏等等问题，这也是我们常常把JS写在页面最底部的原因。

异步加载JS 为了避免加载、执行JS的时候阻塞DOM渲染，我们可以在<script>标签加入defer和 aysnc 来开启异步加载JS，这样就不会阻塞DOM渲染

两者的区别

  • 两者都是异步加载，不会阻塞DOM渲染
  • defer的脚本会按照页面排版的顺序执行。
  • async的脚本加载后会立即执行，执行顺序不确定。
  • 引用B站技术蛋老师的图片更加直观~

# Load 和 DOMContentLoaded 区别

  • Load 事件触发代表页面中的 DOM，CSS，JS，图片已经全部加载完毕。
  • DOMContentLoaded 事件触发代表初始的 HTML 被完全加载和解析，不需要等待 CSS，JS，图片加载。
