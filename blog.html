2022个人年度总结

December 12, 2022 · 13 min read
lavard
Front End Engineer @ Thoughworks
想让这个世界听听自己的意见

学业总结​

双非切图仔的自我救赎

嗨！我是 lavard，笔者来自广东某一本高校, 专业是计算机科学与技术(CS), 方向是前端开发,出生于广东非珠三角最贫穷的地区(广东省 GDP 倒数第一的市区), 通过 读书改变命运 这一想法从我懂事起就从未改变. 直到今日, 我仍然认为 教育 对一个人是最为重要的, 没有正确的教育引导, 社会一切秩序将不复存在.

选择 CS​

2018 年, 刚刚参加完高考的我对专业方向感到迷茫, 似乎自己并没有一技之长, 但是在了解到社会形势以及亲友的推荐后, 我选择了计算机科学与技术(CS)专业。 大一大二都是塞满课表的通识课和计算机基础课，到了大三，我开始感到危机感，开始思考未来的方向， 最终决定选择 web 前端方向， 学习过程中我也从一个什么都不懂，什么都要问的菜鸟逐渐成长为一个程序猿。逐渐如何在互联网上收集信息， 解决问题， 回想那段日子， 这段时光只能用 难熬的青春 来形容吧！ 对未来充满憧憬但又伴随焦虑， 即便是现在这种感觉也依旧存在。

校招之旅​

正式学习前端是在大三下前的暑假，2 个多月的时间我只能从最基础的三件套学起，还没接触到框架。白天看视频学习，边敲代码边做笔记。累了就刷博客以及在牛客网刷面经 以及在 掘金 看总结文。随后返校开始准备秋招，校招期间我已经积累了一定的项目经验以及博客撰写， 但还存在唯一的短板就是算法，虽然前端实际业务中并无涉及过多算法，但算法依然是考察一个程序员逻辑思维的重要途经(其实就是内卷加剧)，所以每天都在坚持刷个几道算法题，不敢说这能使我算法水平提升显著，但对手撕算法的恐惧感确实在逐渐下降。甚至不少笔试都能全部 AC，这给我带来很大的信心。在面试求职的过程中我也在不断总结优劣， 不断改进，因为研发岗的笔试基本都是在牛客或者赛码网（少数使用自己的平台）写题，题型主要是选择题和算法题，时长普遍在一到两个小时。在感觉不满意的情况下，便会刻意在这些平台上练习。而面试无非就是八股文~ 感受就是只要 基础功扎实，便可成为 offer收割机

顺利入职​

首先我要祝贺自己，在 2022 年 6 月，顺利结束了大学四年本科生涯，并在今年 9 月份，在 Thoughworks 开启了前端工程师的旅程。

生活总结​

友情​

本科毕业，我与众多好友迎来了分别，他们的大多数和我一样进入企业开始自己的职业生涯，有的保研或考研至其他学校，这一别不知还要多久才能再次相见，江湖路远，期待我们都有光明的前途。

亲情​

这一年，很明显能够看出父母亲又年老了。我很惭愧，对我的家庭非常惭愧，已经二十多岁的人了，还没有成为家里的顶梁柱，父亲还是需要日夜班两班倒养家，母亲也因长年操劳很早就患了糖尿病。

对我自己说，很抱歉，还没有成为自己想要自己的成为的样子， 有些是自己选择但更多是被迫选择， 但正如 短门先生说 生活是旷野，不是轨道，我有权选择自己的生活 ， 所以挣扎着活下去吧！

技术总结​

开始写 Rust​

22 年因为某些机缘巧合了解到 Rust 并且开始想要学习，但尝试了好几次都没有坚持下去，因为太难了。 不止一次被困难挫败，之所以学习 Rust 而不是其他语言，是因为学习 Rust 能让我更加从底层知识去了解整个计算机体系，而不只是 API 调用工。

Rust 事实上已经成为我最喜欢的编程语言了，Rust 的性能和表达能力都很优秀，就业岗位暂时不是很多意味着还没有很卷，大企业和 linux 对 rust 的关注代表着它的前途，希望身边还没有开始 Rust 的同学马上开始行动。（我就是 Rust 布道者 , 虽然平时我还是在写js）

行业相关​

平时都是比较关注计算机、互联网等领域的资讯, 从中可以了解技术风向和发展趋势以及就业形势等等, 就是近几年，已经看到不少互联网公司大幅裁员的新闻，其中最为恶劣还有应届生裁员等等, 互联网"寒冬"也许正在到来, 但不管如何, 依旧会选择在这行业挣扎. 当然也不是没有想过赚够留学的钱然后润(run)去欧州读 PhD, 但终究是想象不敢付诸于行动, 毕竟国内的才是我最熟悉的.

同时，今年我还见证了许多历史事件,俄乌冲突,上海疫情... 这些事件都充分证明了 人只能被形势所推, 而强者造就形势.

经济下滑​

互联网已经从增长时代 逐渐过渡 到 存量时代, 互联网大面积的裁员便是佐证. 看不过来的撕三方裁应届.

  • B 站、京东 毕业,率先拉起裁员浪潮
  • 杭州 有赞 裁员全部校招应届生
  • 深圳 转转 5 月裁员全部校招生
  • 杭州 税友 6 月份毁约应届生
  • 北京 北森6 月毁约应届生
  • ....

讨论的消亡​

我一直对中文互联网社区有强烈不满， 但是我也不确定这种不满的具体来源。 但可以肯定的是， 中文社区的讨论正在消亡。B 站、知乎、微博等等平台的评论限制审查，不符合氛围的评论直接屏蔽，留下的只是想让我们看到的，讨论区评论区都只是 情绪输出而不是 观点输出 ， 这真是令我失望至极，但又无力改变。

其次是技术社区的堕落，中文技术论坛像 CSDN 抄袭灌水而不加管理的状态以及持续不知多久，长期导致的结构就是中文搜索的质量非常低下，即便使用Google，首页也时常出现以上情况，更有甚者，国内厂商以及某些利益熏心的开发者更是自建内容农场，爬取他人网页聚合，既侵犯他人版权又导致想要真正查询这篇文章的用户浪费时间，这种情况下，技术社区的讨论也会消亡。

2022 年，依旧笼罩新冠疫情的阴影下, 这给平时学习和工作的生活都带来许多不便利, 但生命无价, 希望疫情能早日消散吧!

使用github-action自动化部署Docusaurus

August 5, 2022 · 3 min read
lavard
Front End Engineer @ Thoughworks

什么是github-action?​

简单来说,就是可以启动一个小虚拟机进行一系列自动化操作 ,包括但不限于 打包、部署、推送、定时任务..etc, 本站就是由Github-action自动化部署而来, 更新博客后推送便会自动化构建并部署到github-pages, 本文将记录我搭建的简单过程.

配置​

创建gh-pages分支 (名字任意取,但需要至少2个分支 或者 2个仓库)

Github Pages 会从默认分支（一般是 master/main）或者 gh-pages 分支中提取部署文件（运行 docusaurus build 产生的文件）。文件可以放在根目录，也可以放在 /docs 目录中。 你可以在仓库的 Settings > Pages 处配置。 这个分支会被称作「部署分支」。

docusaurus.config.js

module.exports = {
  // ...
  url: 'https://endiliey.github.io', // 你的网站 URL
  baseUrl: '/',
  projectName: 'endiliey.github.io', // 你的仓库名
  organizationName: 'endiliey',  // 你的组织名 或者 用户名
  trailingSlash: false
  // ...
};

deploy

cmd /C "set "GIT_USER=lalalavard" && yarn deploy"

github-action 脚本​

在.github/workflows/deploy.yml 添加以下内容,没有则自行创建

name: Deploy to GitHub Pages

on:
  push:
    branches:
      - main
    # 如果你想要进一步定义触发条件、路径等，可以查看文档
    # https://docs.github.com/en/actions/using-workflows/workflow-syntax-for-github-actions#on

jobs:
  deploy:
    name: Deploy to GitHub Pages
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v2
      - uses: actions/setup-node@v3
        with:
          node-version: 18
          cache: yarn

      - name: Install dependencies
        run: yarn install --frozen-lockfile
      - name: Build website
        run: yarn build

      # 部署到 GitHub Pages 的热门选择：
      # 文档：https://github.com/peaceiris/actions-gh-pages#%EF%B8%8F-docusaurus
      - name: Deploy to GitHub Pages
        uses: peaceiris/actions-gh-pages@v3
        with:
          github_token: ${{ secrets.GITHUB_TOKEN }}
          # 要发布到 `gh-pages` 分支的构建输出：
          publish_dir: ./build
          # 下面两行会将此次部署 commit 的作者设置为官方的
          # GH-Actions 机器人：
          # https://github.com/actions/checkout/issues/13#issuecomment-724415212
          # 如果不设置这两个字段，GH actions 机器人会被默认使用。
          # 你可以用自己的用户信息替换它们。
          user_name: github-actions[bot]
          user_email: 41898282+github-actions[bot]@users.noreply.github.com

如何高效学习

August 5, 2022 · 5 min read
lavard
Front End Engineer @ Thoughworks

学习方法​

好的学习方法能让人事半功倍, 如果对学习效率不满意, 不妨判断一下自己的学习方法是否适合自己, 是否足够高效. 笔者也是从不会学习 到 逐渐 培养起适合自己的一套高效学习方法, 希望能够帮助大家.

语言学习​

从大学才开始接触编程，第一门编程语言是 C，后续接连学习了 JavaScript、Java、Python、Rust 和 等其他语言。语言不在多而在精, 精通一门语言才是重中之重, 但可以学习其他语言来开拓视野,我也来分享下学习这些语言的想法和感受:

  • C 学习的第一门语言, 从底层出发了解整个计算机系统, 但不好驾驭
  • JavaScript目前吃饭的家伙,浏览器只能跑 JS,没什么好说的,又爱又恨吧~
  • Java 生态最成熟应用最广泛, OOP 特性让团队合作更加高效
  • Python 各种各样的库很多，可以快速验证自己的想法
  • Rust 学习曲线十分陡峭, 不过好在编译器会手把手教我写代码

找准方向​

学习路线​

打好基本功

无论学习什么,一份学习路线的指导是非常有必要的,这样可以让你少走许多弯路,用最短的时间学习最多的知识, 千万不要跳,一个阶段一个阶段的走下去,这是打基本功的好办法, 基础不牢地动山摇.

由于笔者为前端工程师，例子为前端的学习路线

前端学习路线 2022​

  1. HTML+CSS (step1)
  2. JavaScript (step2)
  3. Vue/React/Angular (step3)
  4. Webpack/Vite (step4)
  5. Nuxt.js/Next.js (step5)
  6. Node.js (Nest.js) (step6)
  7. What ever you want(step7)

勤做笔记​

学习应该伴随 输入 和 输出

学习一门新语言 or 新技术 一定要输出!!! 也许你看了某个视频/文章 你觉得你会了, 但是这些并没有让你大脑留下深刻印象, 因为学习需要输入和输出的反馈, 最好的方法就是 边学边做(动手实现并记录笔记)

Notion​

笔记软件我推荐我 Notion, 可以不夸张地说, notion 就是所有笔记应用的终点, 以至于市面上你听到的笔记应用或多或少都带有 notion 的影子.

检验成果​

项目验收

赶紧把你学习的新技术用在项目上吧, 哪怕只是一个简单的 demo, 这也会让你留下更加深刻的印象.

做完项目最好上传到 github 并持续打磨后开源, 如果你是和小伙伴一起完成的那更加好。这不仅让你体会到 git 多人开发的过程，期间你还可以练习如何 pr，如何提交规范的 commit，遇到冲突如何解决，最重要的是你参与了一个开源项目的诞生/贡献！在这项目过程中，你也可以把你学习到的技术运用其中，如果能够获得 1000 个 star，那将是一件了不起的事情！

2022 漫画阅读清单​

今年都在看世界第二自由的漫画家 - 藤本树

作品风格上，因酷爱电影，他创作的漫画分镜具有独特的电影感。藤本对于主角之外的角色并不怎么爱惜，有时候会一通乱杀，加之其出乎意料的剧情风格，他被读者称为“地表第二自由漫画家”。漫画中常常会出现妹控情节 🤗

  • 电锯人
  • 炎拳
  • 再见绘梨
  • Look back
《电锯人》

“现在，要起飞了”，2020年，一位在《少年Jump》首次连载的作家——藤本树的作品《电锯人》掀起了网络上的热烈讨论风潮，该漫画的介绍是十分简单的一句话：“震荡世界的黑暗英雄故事。”然而在短短几个月内，这部作品刷新了我对青年漫画的认识，如同《刃牙》的作者板垣惠介在创作上遵循着“只要作品的角色有魅力那么一定会叫座”的原则，而藤本树在简单介绍了主角电次，一个唯一的伙伴是一条恶魔犬的孤儿，艰难的生活背景和被背叛杀害后像其他热血漫画主角一样的获得了“特殊能力”后，就引入了这个每一幅都有着溢出格子的魅力的角色——玛琪玛，而实际上，这两个角色之间的微妙化学反应，正是剧情向前发展和促使着我向下阅读的动力。

玛琪玛作为一个十分不传统的反派人物，同时也是这部作品某种程度上最重要的角色，其个性专注，冷酷又偏执，这些个性在剧情的发展中慢慢被揭露，而这些个性也可以说完美的被继承到了这个角色直到退场前的最后一刻。但尽管剧情是这部漫画引人入胜的部分，我依旧要说玛琪玛这个角色的成功离不开，甚至是主要归功于藤本树的角色外形设计，一条麻花辫与长鬓姬发，精简干练的西服套装与黑风衣，可以说这个飒爽到近乎英俊的外形让我在玛琪玛出场的第一格就被征服了。 “我喜欢她的外貌和气质，所以我想知道更多这个角色的各种方面。”带着这样的心情和想法，我对她格外关注，当藤本树画出她的特写时，我会在看完对话后不自觉地翻回玛琪玛那一页。有时我会骗自己说是不是那页还有对话或者剧情我没有读完，但我内心知道我只是忍不住想多看一眼这个女人的脸。 就这样，我反而开始注意到了那些我平常没去注意的地方，我不是个有着任何绘画或者电影相关专业知识的人，但随着我被迷住而给与更多注意力于她时，我开始赞叹“原来漫画真的这么厉害”，每一格的构图，玛琪玛近乎完美的身体比例，黑白漫画下光线照在人身上的表现，与这些细节所暗示的，更深层的一些东西，这从根本上颠覆了我对漫画的一些看法。当我看着那有限的笔画与荒诞的故事，藤本树碍于篇幅与时间没画出来但我确信在这一格前我和他都在想象的一个庞大精美又像是B级片的场景在脑中展开时，我的心中升起了一种确信，我正在见证的东西，这就是“浪漫“。

《炎拳》讲述了一个叫阿格尼的男人为向德玛复仇，忍受着不断燃烧的苦痛，一直前行的故事。一路上遇到了形形色色的人，有的似影随身，有的只是匆匆过客。在与他人的交往中，阿格尼的立场随之变化，最终迎来了永久的沉睡。

故事设定在“旧时代”后的 300 余年，即“新时代”。在寒冷的未来，物资极度匮乏，阿格尼与自己的妹妹露娜，作为两位再生能力祝福者，通过不断砍下手为全村人提供赖以生存的食物。来自当时一个由宗教统治的城邦——贝城的德玛，来到了阿格尼所在的村庄强征奴隶，发现村中人食用人肉后便通过自己“将目标燃烧至死”的祝福，杀死了除阿格尼以外的所有村民，包括其亲妹妹露娜。而阿格尼的再生能力过于强大，无法被火焰燃尽，便以一直燃烧的姿态独自存活了下来。露娜临死前的一句“活下去”，成了他当时存在的唯一理由，故事由此开始。

《再见绘梨》

藤本树的漫画从各种短篇到《炎拳》再到《电锯人》，将会是我一生最爱的漫画艺术作品。 特别是《电锯人》中黑暗恶魔出场的画面，我曾向朋友发疯一样的表达过：这一页应该放大裱起来，展览在美术馆里，这是艺术。 不光画作和剧情本身就像艺术一样的令人迷惘，从玛奇玛和狗，打雪仗，到拍电影，成为树，藤本树的漫画中总会透露出他对于艺术看法的探讨。 《Look Back》给出了藤本树自己的答案：因为无奈但不可名状的热爱，而《永别了绘梨》则是观众的答案，就像《肠子》的引言爱伦坡的文字一样：不会引起反感的怪异。虚无，但是有爆炸！唯有爆炸不可辜负！燃烧一切的火焰！燃烧的也是虚无！ 《永别了绘梨》的故事里，优太的妈妈是虚无的妈妈，电影造就了她的美丽人生和如爆炸般绚丽的死亡。而优太疑惑，妈妈才是电影的主角不是吗，妈妈是美的不是吗。观众为什么不喜欢。爸爸也是虚无的爸爸，他拍了关于反派妈妈的电影，就像迪士尼坏人们一样。 绘梨她不好也不坏，她是俗套的看点，但仍然是引进的幻想元素。失忆的吸血鬼和她离世的爱人，一场更绚丽宏大的爆炸。 虽然绘梨是吸血鬼这件事看上去很扯，只不过从抖动画面和正常画面的区别来看，正常的画面多半都是演的。所以绘梨是吸血鬼虽然可能是真的，但失忆，反复看那部电影，最后的爆炸，都应该是演的。（但是想到是藤本树，感觉什么都会很合理） 漫画的最后，绘梨说： 即便我会无数次的忘记你，但我仍会无数次的将你记起，这不是一件非常美妙的事情吗？ 她记起的是谁，是电影里的优太吗？是那个跟她接吻，给她看自己大便视频的优太吗？记起一个美好的优太，当然会是一件美妙的事情。 那我们看完后记起的是谁，是电影里的优太吗？是会拿着相机记录几百个小时，以电影为梦想却仍然在拍烂电影的优太吗？是被妈妈抛弃，被绘梨抛弃，但仍然在引用爆炸作为幻想元素的优太吗？是在进步的优太？还是没有天分的优太？ 说起来，我们真的见过真正的优太吗？ 但，我记住了爆炸。在一个人死亡时有爆炸。我看见了拍摄这部电影的藤本树。 《永别了绘梨》的故事，“那是魏提尔先生所谓的摄影机后面的摄影机后面的摄影机”。那应该是“摄影的时刻：真相的真相”。那是爆炸后面的爆炸，是将虚无具象的虚无。 整件事就像是一个苹果的诅咒一样，你一旦爱上了藤本树的虚无，遍很难从现实中寻找到可以弥补的巨物。可以填补藤本树虚无的，永远都会是下一部藤本树的虚无。 《永别了绘梨》无疑是好作品，因为我甚至没有办法描绘看完后我获得了什么。无法复刻的情感，无法抒发的喜爱，无法忽视的诡异，这些唯一带来的满足感，每个人都察觉到了什么东西出现了，又消散不去，因而每个人都在享受艺术。因而每个人都在拍摄藤本树。 另：如果你想去看《肠子》，请慎重考虑，真的有一点恶熏。或者可以考虑替换为继续刷《炎拳》，学习它，领悟它，这是每一个藤本树爱好者都应该具备的炎拳藤本树的技能。

《Look back》这部短篇漫画以两位喜爱画漫画的少女为主角，藤本树在这部短篇中将他的特色发挥得淋漓尽致，无论是分镜和还是剧情都充满了他的个人风格。

为何选择 docusaurus?​

笔者之前的博客一开始是使用hexo搭建的, 后来学习了vue,偶然逛知乎发现尤大写了个项目,没错就是vuepress, 一使用便爱上了,后来随着 Vue3和Vite的诞生又迁移至Vitepress, 对Vite闪电般的构建速度十分满意, 但Vitepress尤大交给了 core team 的日本开发者,但这开发者实在太懒了, 存在许多Bug , 加上在工作中更多地的使用React, 最终决定迁移至docusaurus

各静态网站生成器对比
  • VitePress
  • docusaurus
  • Next.js
VitePress十分强大,借助于Vue3的强大表达能力和Vite的闪电般构建速度, 一度使我想要为其贡献代码 , 但现在的工作技术栈和Vue相关不多, 加上我太懒了😊

升降桌​

站立交替办公， 远离腰间盘突出、痔疮、静脉曲张！

久坐的危害
  1. 长期久坐会减慢血液循环的速度, 对身体的健康有很大的影响。尤其是坐姿不当，不仅会引起驼背，还会造成骨质增生。
  2. 长期久坐会造成胃肠蠕动迟缓，会减少消化液的分泌，时间久了就会引发消化不良、食欲不振或者脘腹饱胀等病症。
  3. 长期久坐会长痔疮，久坐导致腹部血液的流动速度减慢，下肢静脉血液不能得到回流。长时间这样，直肠静脉丛特别容易发生曲张，造成血液淤积，形成了静脉团，也就是 痔疮。

屏幕挂灯​

屏幕挂灯用处
  1. 补光。且光照打到屏幕上不会反射，同时照亮桌面周围环境，无蓝光危害，无可视频闪，补充光照又不伤眼。
  2. 简洁。屏幕挂灯无需占用桌面额外空间，只需要挂到屏幕上，利用屏幕或者主机 usb 口供电

之前尝试过双屏，但需要来回扭加上多台设备的繁琐，逛油管的时候发现许多博主都在用这款 LG 的带鱼屏，5120x2160 的分辨率,98%的 P3 色域，HDR600 认证，支持 85W 反向供电，第一眼就种草了！

Windows​

# 1. 安装scoop (windows的第三方包管理器,类似Mac的homebrew)

iwr -useb get.scoop.sh | iex

# 2. 安装neofetch

scoop install neofetch

# 3. 获取系统信息

neofetch

Mac​

  1. Run brew install neofetch after installing Homebrew
  2. Run neofetch

Linux​

💡 点击访问 screenFetch 源码仓库 , 自行对应使用的 Linux 发行版 安装 screenFetch

# 以 ubuntu/debian 为例, 安装screenfetch

sudo apt-get install screenfetch

# 运行screenfetch

screenfetch

QQ群管理机器人-真寻

Front End Engineer @ Thoughworks

最近接触了一款可玩性非常高的 QQ 群聊框架-真寻, 给 QQ 群聊带来许多乐趣, 在这里简单记录下我的使用经验.感兴趣的可以探索仓库 zhenxun_bot

简介​

一款开源的 QQ 群聊管理应用

此项目基于 Nonebot2 和 go-cqhttp 开发，以 postgresql 作为数据库的 QQ 群娱乐机器人

技术来源​

#使用以 OneBot 协议为准的框架

基于  Mrs4s / go-cqhttp (opens new window)(没错，超好用的无头QQ)  作为基础框架后端使用  nonebot / nonebot2 (opens new window)(反正好用就是了)  为后端框架开发

数据库​

目前仅仅只支持了Postgresql关系型数据库，ORM 主要使用  python-gino / gino(opens new window)

安装部署​

安装 go-cqhttp​

如何安装？

  1. 从Mrs4s / go-cqhttp (opens new window)的 Release 中下载与你系统对应的最新版本

  2. 首次运行会让你选择通信方式，选择  3 (反向 Websocket 通信)，会生成一个配置文件config.yml

  3. 打开config.yml文件，将 uin 修改为 bot 账号

  4. 在config.yml文件中将universal: ws://your_websocket_universal.server修改为universal: ws://127.0.0.1:8080/onebot/v11/ws/

  1. 重启 go-cqhttp 后扫码登录即可

因本项目使用了语音文件等，该音频需要依赖 ffmpeg

sudo apt install -y ffmpeg #Ubuntu：

没有 ws-reverse 怎么办？第二步的配置文件选错了，删除config.yml重新生成即可
